# Product Requirements Document (PRD)
*First document should be write when start new project*. Selayaknya ketika kamu menulis 'Kebutuhan Fungsional' dan 'Kebutuhan Non Fungsional' misalkan di skripsi atau tugas akhir ketika mengembangkan sebuah aplikasi. *Honestly, I am not sure yet what's best practice when writing this document, like what's section should be required to include? Based on my experience, i will start to define Tech Stack first, then the problem or why we built this, and also what's [[1763225591-minimum-viable-product-mvp|MVP]] should be to finish before over-engineering happens*. Pengambilan keputusan untuk Tech Stack ini didasari oleh pengalaman. *I am mainly used PHP or [[1763224944-laravel|Laravel]] as a framework but i know like any ecosystem in other language like in NodeJS with BullMQ for queueing things* . Tapi penggunaan Tech Stack tadi menyesuaikan bagaimana *scope* aplikasi tersebut. Misalkan, saya tidak akan menggunakan PHP atau di sini [[1763224944-laravel|Laravel]] jika aplikasi tersebut hanya bertugas untuk *web automation* saja. Ya, memang ada cara atau dengan PHP bisa kita lakukan untuk *web automation* ini, tapi kendalanya ada di pertanyaan berikut "*You sure use Laravel for just tiny things?*". Kecuali jika kita ingin membangun *project* *web automation* tadi yang dapat di kelola atau memiliki banyak fitur lainnya.
