# Product Requirements Document (PRD)
*First document should be write when start new project*. Selayaknya ketika kamu menulis 'Kebutuhan Fungsional' dan 'Kebutuhan Non Fungsional' misalkan di skripsi atau tugas akhir ketika mengembangkan sebuah aplikasi. *Honestly, I am not sure yet what's best practice when writing this document, like what's section should be required to include? Based on my experience, i will start to define [[1763281621-tech-stack|Tech Stack]] first, then the problem or why we built this, and also what's [[1763225591-minimum-viable-product-mvp|MVP]] should be to finish before over-engineering happens*.
